Java 必知必会——基本数据类型

发表于 2021-06-19 分类于 Java

Java 是一种强类型语言，也就是说必须为每一个变量声明一种类型。

在 Java 中，一共有 8 种基本类型，其中有 4 种整形、2 种浮点类型、1种表示 Unicode 编码的字符单元类型 char 和 1 种表示真值的 boolean 类型。

1. 整型

整型用于表示没有小数部分的数值，允许负数的出现。

取值范围

类型 存储需求 取值范围
int 4 字节 -21 4748 3648 ~ 21 4749 3647 （超过20亿）
short 2 字节 -3 2768 ~ 32767
long 8 字节 - 922 3372 0368 5477 5808 ~922 3372 0368 5477 5807（超过920万兆）
byte 1 字节 -128 ~ 127

长整型（long）数值后有一个后缀 L 或 l，如 123456L。

十六进制数值前有一个前缀 0X 或 0x，如 0xCAFE。

Java 7 开始，加上前缀 0B 或 0b 就可以表示二进制数，如 0b001 即 9。

同时，可以为数字字面量加下划线，如用 1_000_000 （或 0b1111_0100_0010_0100_0000）表示一百万。这些下划线仅为易读性考虑，编译器会自动去除。

注：Java 中，没有任何无符号形式的 int 、long、short 或 byte 类型。

2. 浮点类型

浮点类型用于表示小数部分的数值。

取值范围

类型 存储需求 取值范围
float 4 字节 大约 ± 3.40282347E+38F（有效位数 6~7 位）
double 8 字节 大约 ± 1.79769313486231570E+308（有效位数 15 位）

double 类型的数值精度是 float 类型的两倍，因此也被称为双精度数值。

float 类型的数值有一个后缀 F 或 f，如 3.14F，无后缀 F 的浮点类型默认为 double 类型。当然也可以在浮点类型添加后缀 D 或 d。

特殊说明

浮点类型会出现舍入误差，如 2.0 - 1.1 = 0.8999999999999999，而不是 0.9。

这种舍入误差的主要原因是浮点数值采用二进制系统表示，而二进制系统无法精确地表示分数 1/10，类似于十进制无法精确表示分数 1/3如果数值计算中不允许出现任何这种误差，应该使用 BigDecimal 类。

3. char类型

char 类型原本用于表示单个字符。

但如今，有些 Unicode 字符可以用一个 char 值表示，另外一些 Unicode 字符需要两个 char值。

char 类型的字面量值要用单引号括住。如 ‘A’ 是编码值 65 所对应的字符常量。

它与 “A” 不同， “A” 是包含一个字符 A 的字符串。

char 类型的值可以表示十六进制，其范围从 \u0000 到 \Uffff。

除了转义字符 \u 外，还有一些用于表示特殊字符的转义序列。

这些转义序列都可以出现在加引号的字符字面量或字符串中。如 ‘\u2122’ 或 ‘hello\n’。

转义序列 \u 还可以出现在加引号的字符常量或自负床之外（其他转义序列不可以）。

例如 public static void main(String\u005B\u005D args)，这符合语法规则，因为 \u005B、 \u005D 分别对应 [ 、 ]。

特殊字符的转义序列

转义序列 名称 Unicode 值
\b 退格 \u0008
\t 制表 \u0009
\n 换行 \u000a
\r 回车 \u000d
\ “ 双引号 \u0022
\ ‘ 单引号 \u0027
\ \ 反斜杠 \u005c

特殊说明

Unicode 转义字符会在解析代码前处理，如 \u0022+\u0022 会被解析为 ""+""。

4. boolean 类型

boolean（布尔）类型有两个值：false 和 true ，用来判断逻辑条件。

注：整型值和布尔值之间不能相互转换。
